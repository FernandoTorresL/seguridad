Introducción al Pentesting

El Pentesting se define como un conjunto de técnicas de hacking que ayudan a encontrar alguna vulnerabilidad en un sistema informático con el fin de obtener el acceso a el.

El objetivo principal del Pentesting es determinar la seguridad de un sistema, habitualmente esta serie de técnicas se utilizan para evaluar el cumplimiento de las políticas de seguridad de una empresa y así determinar la capacidad que tiene dicha organización para identificar y responder a problemas de seguridad.

Algo muy curioso de un test de penetración es que no solo comprende el análisis de dispositivos digitales sino también de dispositivos físicos, incluso el análisis del factor humano por medio de la Ingeniería social.

Esta serie de técnicas tienen sus fases:

  • Reconocimiento: Es la recopilación de información (desde nombres, direcciones de correo, direcciones IP y toda aquella que se pueda obtener) sobre el objetivo antes de iniciar la prueba.
  • Escaneo: Es la forma de utilizar la información obtenida anteriormente para encontrar posibles formas de ataques. En esta etapa se involucran puertos y servicios con el fin de ser escaneados para encontrar huecos de seguridad y así definir la ruta de ataque.
  • Explotación: Este punto es clave, ya que en el se obtiene toda la información posible referentes a los usuarios, a los nombres de los equipos, a los servicios de red, entre otros.
  • Manteniendo el acceso: Al lograr tener acceso al sistema, se debe lograr mantener el sistema comprometido con el fin de tenerlo siempre a disposición del atacante.
  • Borrado de huellas: Este punto es uno de los más importantes, ya que es el proceso que se realiza para eliminar cualquier rastro que comprometa al atacante.

El Pentesting puede ser o no ético dependiendo su fin, si es para el bien o para el mal. La metodología de un Pentest se divide entre: BlackBox, WhiteBox y GrayBox.

En Ciberseguridad un vector de ataque es una ruta o camino que utiliza un atacante para obtener acceso al sistema informático, en otras palabras, son los pasos a seguir del Pentester para materializar su ataque. Algunos vectores de ataque son:

  • APT (Amenaza avanzada persistente): Este tipo de ataques es uno de mayor gravedad a los habituales, este posee características que hacen que sus efectos dañinos sean más fuertes. El atacante que usa este tipo de vector de ataque es muy paciente, mucho más que el atacante promedio.
  • Botnet: En este tipo de ataque se usan virus troyanos especiales para violar la seguridad de las computadoras de varios usuarios, así de esta forma tomar el control de cada una de ellas y organizarlas en una red de manera que el atacante pueda administrarlas de forma remota. La palabra Botnet se forma a partir de las palabras "robot" y "red".
  • Ataques internos: Este tipo de ataque son los más comunes (aproximadamente el 80%) y son los que más impacto pueden generar en una infraestructura ya que por estar en el interior se es más fácil acceder a la red interna y desde allí atacar.
  • Amenazas Móviles: Este tipo de ataque parte de la tendencia de llevar consigo un dispositivo móvil y por medio de ellos acceder a la información, a la red de una empresa.

Algunos términos importantes que se deben conocer son:

  • ¿Qué es un activo?: Es todo lo valioso para una empresa.
  • ¿Qué es una amenaza?: Es una falla que puede ocurrir en una empresa, incluso fallas que existen pero no se conocen.
  • ¿Qué es una vulnerabilidad?: Es una debilidad de un sistema informático que puede ser utilizada para causar daños.
  • ¿Qué es un riesgo?: Es un proceso que comprende la identificación de las vulnerabilidades y amenazas dando lugar a un ataque.
  • ¿Qué es un exploit?: Es el proceso de tomar las fallas que existe en el sistema y sacarle provecho.
